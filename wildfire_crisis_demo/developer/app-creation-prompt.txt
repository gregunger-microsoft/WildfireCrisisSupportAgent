DATA MODEL (Pydantic v2)
Define Pydantic v2 models:
Input:
- WildfireIncidentBundle:
  - incident_summary: str
  - weather_feed: list[WeatherUpdate]
  - fire_behavior: list[FireBehaviorUpdate] (perimeter, rate_of_spread, containment estimates)
  - incident_feed: list[IncidentUpdate] (911 surge, new starts, spot fires)
  - road_closures: list[RoadClosure]
  - evacuation: list[EvacuationUpdate]
  - shelters: list[ShelterStatus]
  - utilities: list[UtilityUpdate] (power shutoff, comms degradation)
  - resources: list[ResourceStatus]
  - constraints: Constraints
  - air_quality: list[AirQualityUpdate]

Core:
- TraceEvent:
  - timestamp_utc: datetime
  - kind: Literal["INCIDENT","AGENT_EXECUTION","POLICY","ERROR"]
  - event_type: str (e.g., "WIND_SHIFT", "EVAC_ORDER", "AGENT_START", "AGENT_END", "SUPERVISOR_BLOCK")
  - source: str (e.g., "WEATHER", "EVAC", "ROAD", "SHELTER", "UTILITIES", "AGENT")
  - summary: str
  - confidence: int 0–100 (or None only if kind="AGENT_EXECUTION" and you document why)
  - citations: list[str]
  - details: dict (strictly JSON-serializable)
  - stage_name: Optional[str]
  - agent_name: Optional[str]
  - duration_ms: Optional[int]
  - status: Optional[Literal["ok","retry","blocked","failed"]]

Domain outputs:
- IncidentSnapshot (DataIngest output)
- SitRep (SitRep output)
- ResourcePlan (ResourceAllocator output)
- SupervisorReview (Supervisor output)
- CrisisActionBrief (final output):
  - overview, risks[], COAs (3), resource_plan_summary, verification_checklist,
    public_message, internal_message, metadata (confidence, citations, correlation_id)
